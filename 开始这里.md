# 开始这里

本诊断包只读取系统类型、常用工具版本、代理/证书变量是否存在，以及几个官方
地址是否可达。它不会读取变量值、密码、API Key、访问令牌、Codex 认证文件、
项目代码或文件内容，也不会安装、卸载或修改任何软件。

## 第一步：运行对应脚本

### Windows 10/11

1. 解压 ZIP。
2. 打开 PowerShell。
3. 进入解压后的文件夹。
4. 运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\collect-diagnostics.ps1
```

### macOS / Linux / WSL2

打开终端，进入解压后的文件夹，运行：

```bash
bash collect-diagnostics.sh
```

脚本会在当前目录生成 `codex-safe-report.txt`。

## 第二步：人工确认

用文本编辑器打开报告，确认其中没有以下内容：

- 姓名、邮箱、公司名、主机名或项目名；
- 用户目录、工作目录、内网地址、代理服务器地址；
- 密码、验证码、Cookie、API Key 或访问令牌；
- 源代码、配置文件正文或日志全文。

发现任何不想分享的内容，直接删除对应行。不要额外发送 `auth.json`、
`config.toml`、`.env`、终端完整历史或截图中的密钥。

## 第三步：提交

把以下信息通过购买平台的私信发送：

1. `codex-safe-report.txt` 的文本内容；
2. 你想完成的结果，例如“安装后能运行 Codex CLI”；
3. 卡住时屏幕上最后一条错误信息，先删掉个人路径和账号信息；
4. 问题最早出现的大致时间，以及此前是否能够正常使用。

交付内容为一份诊断报告和一轮补充判断。不要发送任何账号凭据；需要登录时，
只能由你本人在自己的设备上完成。
