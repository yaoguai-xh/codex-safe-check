Paid Diagnosis Flow

The script is free. Payment covers a manual diagnosis report based on a redacted collector output, prioritized fix steps, and one follow-up review.

The payment link is not live yet. Do not send diagnostic reports through public GitHub issues. Public issues are only for safe documentation or collector feedback. Launch notifications are posted in the public paid diagnosis status thread.

What the paid report includes

  • A written diagnosis of the likely failure layer: install, PATH, CLI version, login, account boundary, proxy, custom CA, network reachability, or local state.
  • Fix steps ordered by risk and likelihood, with expected result and stop condition for each step.
  • One follow-up review after you run the requested checks.

What to send after payment

  • Your main symptom in one sentence.
  • The full text of codex-safe-report.txt after manual privacy review.
  • One redacted error line if it is necessary to understand the failure.

Do not send

Do not send passwords, one-time codes, API keys, access tokens, cookies, identity documents, auth.json, full logs, full configuration files, source code, screenshots containing account details, usernames, hostnames, working directories, or proxy addresses.
